File Upload Form (Disk)
File upload, HTTP request body saved to DISK
Use the form below to upload relatively big files to the web server. The controller used here specifies that the HTTP body should be temporary stored in the DISK partition of the web server. For this reason the maximum HTTP body size is <%=attr_max_upload_size%> Kbytes. Use the ELYSIAN_MAX_HTTP_BODY_SIZE_KB_DISK configuration option to increase the maximum allowed HTTP body size.

HTTP controllers that store the HTTP body in the DISK partition (SD Card or HDD) are suitable for bigger HTTP requests etc when there is a need to upload .mp3 or .pdf files.

For an example showing the same form submission, but storing the HTTP body to the RAM memory instead of DISK partition, refer to the "File Upload Form" example, whose controller saves the HTTP body to the RAM memory.
