File Upload Form (Disk)
File upload, HTTP request body saved to RAM
Use the form below to upload a small file to the web server. The controller used here specifies that the HTTP body should be temporary stored in the RAM memory of the web server. For this reason the maximum HTTP body size is <%=attr_max_upload_size%> Kbytes. Use the ELYSIAN_MAX_HTTP_BODY_SIZE_KB_RAM and ELYSIAN_MAX_MEMORY_USAGE_KB configuration options to increase the maximum allowed HTTP body size.

HTTP controllers that store the HTTP body in the RAM memory are suitable for small HTTP requests etc for small HTML forms with configuration options.

If there is a need to upload files of bigger size (etc .mp3 or .pdf files), refer to the "File Upload Form (Disk)" example, whose controller saves the HTTP body to the DISK partition (SD Card or HDD).
