HTTP persistent connections and pipelining
HTTP persistent connections, also called HTTP keep-alive, is the idea of using the same connection to send and receive multiple HTTP requests/responses, as opposed to opening a new connection for every single request/response pair.

HTTP pipelining is a feature of persistent connections. It allows the Browser to send a new request on the same connection without waiting for the response of the previous one. These back to back HTTP requests result in a dramatic improvement in page loading times.

Both features are used to minimize transaction delays for the approximately 10 HTTP Requests triggered from this page.
