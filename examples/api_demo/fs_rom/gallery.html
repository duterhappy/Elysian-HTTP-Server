HTTP pipelining, keep-alive connections and multi-client support
Despite of the single thread implementation and due to a number of implemented HTTP features and architecture, Elysian offers unique perfomance when dealing with concurrent HTTP requests.

Except from core HTTP features like pipelining and keep-alive connections the internal state machine is designed to overcome temporary memory or network unavailability issues which is a typicall case on memory contrained embedded devices.

This is achieved by means of the internal exponential backoff mechanism that re-schedules temporary failed operations on periods of high load.
