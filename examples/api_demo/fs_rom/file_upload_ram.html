File upload, HTTP request body temporary saved to RAM
HTTP file upload is the process that enables an HTTP client to send files to the Web Server. In the same way there are HTML form elements that enable text input, there are also text elements for choosing a file.

There are only a couple of real differences. The input type of the upload is 'file' rather than 'text'. The 'enctype' of the form is 'multipart/form-data'. The multipart content type is an internet standard for sending lots of different types of information over a single HTTP connection as one bundle.

The uploaded data should be temporary stored by the Web Server before given to the application layer for further analysis. Elysian Web Server can store these kind of data either to RAM or to the EXT(ernal) (SD Card, USB or HDD drive) memory device.

The MVC controller implemented here stores them to RAM. The maximum allowed upload size can be configured using the ELYSIAN_MAX_HTTP_BODY_SIZE_KB_RAM definition. Current value is <%=attr_max_upload_size%> Kbytes. ELYSIAN_MAX_MEMORY_USAGE_KB is another option controlling the maximum allowed RAM usage and should be configured accordingly.

For an implementation allowing bigger file uploads refer to the "File Upload (EXT)" example, that uses the EXT(ernal) memory device as temporary storage medium.
