File upload, HTTP request body saved to the "EXT(ernal)" (Hard Disk) memory device
HTTP file upload is the process that enables an HTTP client to send files to the Web Server. In the same way there are HTML form elements that enable text input, there are also text elements for choosing a file.

The input type of the upload is 'file' rather than 'text'. The 'enctype' of the form is 'multipart/form-data'. The multipart content type is an internet standard for sending lots of different types of information over a single HTTP connection as one bundle.

The uploaded data should be temporary stored by the Web Server before given to the application layer for further analysis. Elysian Web Server can store these kind of data either to RAM or to the EXT(ernal) (SD Card, USB or HDD drive) memory device.

The MVC controller implemented here stores them to the EXT(ernal) memory device -- The Hard Disk. The maximum allowed upload size can be configured using the ELYSIAN_MAX_HTTP_BODY_SIZE_KB_EXT definition. Current value is <%=attr_max_upload_size%> Kbytes.

For an implementation using the RAM memory device as temporary storage medium, refer to the "File Upload (RAM)" example.
